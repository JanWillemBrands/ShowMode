# ShowMode

## What the app does

ShowMode is a kiosk-style display app for an iPad mini 3 (iOS 12.5.8) that runs unattended in portrait mode. It shows:

- **Background**: Rotating full-screen nature photographs (Dolomites and Venice) from Unsplash, crossfading every 30 seconds.
- **Clock & date**: Current time and date in the Europe/Rome timezone.
- **Weather**: Current conditions and 6-day forecast for Padua, Italy, using the Open-Meteo API (no API key required).
- **Events feed**: Upcoming concerts, opera, and cultural events in Venice and the Veneto region. Events rotate in pages of 6 cards every 20 seconds. Tapping a card opens the event's website in a WKWebView overlay.

Events come from two sources:
1. An RSS feed hosted on GitHub Pages (`veneto-events/docs/events.rss`), generated by Python scrapers that run via GitHub Actions. Covers: Bru Zane, Barco Teatro, OPV (Orchestra di Padova e del Veneto), and Pollini.
2. La Fenice (Teatro La Fenice, Venice) is scraped directly by the iPad app at runtime, because Cloudflare blocks GitHub Actions' datacenter IPs but allows the iPad's residential IP.

Events from both sources are deduplicated by URL (La Fenice items take priority) and merged into a single display list.

## Technical choices and why

### UIKit, not SwiftUI
The target device runs iOS 12.5.8. SwiftUI requires iOS 13+.

### Dual-Xcode compilation
Swift source must compile under **both** Xcode 26 (simulator, for development) and Xcode 14.3.1's `swiftc` CLI (for the iPad binary, targeting `arm64-apple-ios11.0`). Xcode 14.3.1's GUI doesn't launch on macOS Tahoe, but its CLI tools work. Xcode 26 cannot produce iOS 12-compatible binaries. This means: no SwiftUI, no Combine, no Swift concurrency (`async`/`await`), no `#available` guards above iOS 12, and no SPM dependencies.

### `build-ipad.sh` instead of Xcode build system
Xcode 14.3.1's project GUI is unusable on modern macOS, so a shell script drives `swiftc` directly. The script generates `Info.plist`, creates `PkgInfo`, ad-hoc signs, packages the `.ipa`, and validates the result. It runs automatically as a post-build action in the Xcode 26 scheme.

### Sideloadly for installation
Standard code signing and `ideviceinstaller` don't work for this device/OS combination. Sideloadly handles the re-signing and installation. The IPA must include `PkgInfo`, ad-hoc code signature, and full platform metadata (`MinimumOSVersion`, `DTPlatformName`, `DTSDKName`, `CFBundleSupportedPlatforms`) or Sideloadly rejects it.

### No dependencies
No CocoaPods, SPM, or Carthage. Everything is hand-rolled: RSS parsing (`XMLParser`), HTML scraping (string scanning), weather decoding (`JSONSerialization`), image loading (`URLSession`). This avoids compatibility issues with the dual-Xcode setup and keeps the binary small.

### La Fenice scraped on-device
Cloudflare blocks GitHub Actions IPs from `teatrolafenice.it`. Rather than using headless browsers or proxy workarounds, the iPad scrapes the site directly â€” its residential IP passes Cloudflare's checks. The scraper parses the HTML calendar page using string scanning for `data-list-id` date markers and CSS class-based content extraction.

### GitHub Actions RSS feed for everything else
The other venue websites don't use Cloudflare and have simpler HTML. Python scrapers (`requests` + `re`) run on a schedule via GitHub Actions, generate a combined RSS feed, and publish it to GitHub Pages. The feed includes retry logic, event filtering (excludes seminars, workshops, tastings), and preservation of events from failed scrapers or non-scraped sources.

### AppDelegate with window property (no SceneDelegate)
iOS 12 predates the scene lifecycle. The app uses a traditional `UIApplicationDelegate` with a `UIWindow` property.

## Code style
- **Naming**: PascalCase for types, camelCase for properties/methods
- **Properties**: `let` for constants, `private var` for mutable state
- **Formatting**: 4-space indentation
- **Types**: Leverage Swift's strong type system, avoid force unwrapping
- **Comments**: Only for non-obvious logic
- **Testing**: Use the Testing framework for unit tests, XCUIAutomation for UI tests
